# 9. Dangerous Operations

```text
Dangerous Operations
====================
1. Reinstall Default Site
2. Reinstall Another Site
3. Drop Default Site
4. Drop Another Site
5. Restore Over Existing Site
6. Force Restart Production Services
0. Back
```

### Rules

Every action here must:

* ask for confirmation
* ask again with typed confirmation
* optionally require site name re-entry

### Safer confirmation style

```text
This operation is destructive.
Type the site name to continue:
```

# Recommended hidden helper actions

These do not have to appear as menu items, but your script should have functions for them.

## Internal helpers

* select_site
* select_app
* confirm_action
* confirm_dangerous_action
* auto_backup_if_enabled
* detect_production_mode
* restart_if_needed
* log_action
* show_success
* show_error

These will make the script much cleaner.

---

# Best user flow inside the script

When setup reaches bench stage:

```text
Bench setup is ready.
What do you want to do today?

1. Run default site setup flow
2. Open Bench Operations
3. Continue without bench operations
0. Exit
```

Then if they choose Bench Operations, show the full tree above.
