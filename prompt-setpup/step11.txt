
# Step 11: Bench Operations

## What Step 11 is

This is the **interactive Bench control center** inside your script.

Everything from Steps 1–10 prepares the server, bench, site, apps, production, and HTTPS.

Then **Step 11** becomes the part that asks:

```text
What do you want to do today?
```

and lets the operator run bench-related tasks on demand.

So Step 11 is not a one-time setup step like the others.
It is the **operational menu layer** for ongoing use.

---

## What Step 11 should achieve

By the end of Step 11, your script should provide an interactive menu that can:

* manage the bench itself
* manage apps
* manage the default site and other sites
* handle backups and restores
* run updates, upgrades, and maintenance
* inspect and control scheduler/workers/jobs
* manage production services
* run diagnostics
* safely gate dangerous actions

That matches the structure in your uploaded menu tree. 

---

# Step 11 structure

## Entry flow

When the script reaches this step, it should show something like:

```text
Bench setup is ready.
What do you want to do today?

1. Run default site setup flow
2. Open Bench Operations
3. Continue without bench operations
0. Exit
```

That flow is already part of the menu tree you uploaded.
